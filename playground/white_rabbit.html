Searching Algorithms

Linear/Sequential Search
Running Time: O(n)
  • It sequentially checks each element of the list.
  • Start from left-most element.
  • Compare one-by-one.

Binary Search
Running Time: O(Log(n))
  • Search algorithm for Sorted Arrays!
  • Target value is compared with the middle element of the array.
  • If they are not equal and If target smaller than middle element, right half is eliminated.
  • If they are not equal and If target greater than middle element, left half is eliminated.
  • Search continues on the remaining half.

Sorting Algorithms

Selection Sort
Time Complexity: O(n2)
Auxiliary Space: O(1)
  • Worst Case: Array is reverse sorted.
  • Find minimum element in the array.
  • Swap it with first element.
  • Repeat process in remaining part.
  • Thus, subarray is already sorted and remaining part is unsorted.

Bubble Sort
Worst and Average Case Time Complexity: O(n2)
Best Case Time Complexity: O(n)
Auxiliary Space: O(1)
  • Worst Case: Array is reverse sorted.
  • Compare elements n and n+1.
  • Swap them if n+1 smaller than nth element.
  • Repeat process until there is no swap within whole pass.

Insertion Sort
Worst and Average Case Time Complexity: O(n2)
Best Case Time Complexity: O(n)
Auxiliary Space: O(1)
  • Pass from 1th element to nth
  • Compare current element with its predecessor.
  • If current element smaller than its predecessor, compare it until find smaller element.
  • When find a smaller element than current one, slide larger ones and put current one here.

Merge Sort
Worst and Average Case Time Complexity: O(nLog(n))
Best Case Time Complexity: O(nLog(n))
  • Best case time complexity is nLog(n) because merge sort always divides the array into two halves and take O(n) time to merge two halves.
  • Divide and Conquer algorithm.
  • Divide array two halves.
  • If two halves are already sorted, whole array can be sorted by merging them.
  • Therefore, divide array recursively and merge them.

Quick Sort
Worst Case Time Complexity: O(n2)
Best Case Time Complexity: O(nLog(n))
Average Case Time Complexity: O(nLog(n))
  • Divide and Conquer algorithm.
  • Pick an element as a Pivot.
  • Put smaller elements on left side of Pivot and greater elements on right side of Pivot.
  • Pick a new Pivot on that left or right side.
  • Repeat process recursively.

Heap Sort
Time Complexity: O(nLog(n))
  • Sorting by using Binary Heap data structure.
  • Binary heap is a complete binary tree which elements are sorted.
  • By using binary heap root node(Largest element) determine last element of unsorted array.
  • Reduce heap size by 1.
  • Repeat process until heap size is 0.

Radix Sort
Time Complexity: O(nLogb(n))
  • Sort array by using counting sort.
  • Distribute elements into buckets according to their radix.
  • For elements with more than one significant digit.
  • This bucketing process is repeated for each digit.
  • Until all digits are considered, previous orders should be preserved.
